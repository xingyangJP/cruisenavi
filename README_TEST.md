# RideLane テスト手順
バージョン: 1.0.100  
更新日: 2026-02-28

## 1. 前提
- Xcode 17+
- iOS シミュレータ
- `Config/Weather.plist` 設定済み（詳細は `docs/API.md`）

## 2. ビルド
```bash
cd /Users/xingyang/cruisenavi
xcodebuild -project SeaNavi/RideLane.xcodeproj -scheme SeaNavi -destination 'generic/platform=iOS Simulator' build
```

## 3. 自動テスト
```bash
cd /Users/xingyang/cruisenavi
xcodebuild -project SeaNavi/RideLane.xcodeproj -scheme SeaNavi -destination 'platform=iOS Simulator,name=iPhone 17,OS=26.2' test
```
## 4. 手動確認
1. アプリ起動で Home 表示、下部中央バージョンが `ver1.0.100` になっている
2. Home の現在地地図がタイトルなし・角丸なしで横幅いっぱい表示になっている
3. Home 地図の上部中央に `splash` ロゴが表示される
4. Home 地図下に `ルートポイント` / `実GPS` / `GPSアクティブ` などのステータス文言が表示されていない
5. Home の `ver1.0.100` 表示が中央寄せになっている
6. 初回起動時にウォークスルー（5ステップ）が表示される
7. シミュレーターではウォークスルー完了後も再起動で再表示される（検証用挙動）
8. 実機ではウォークスルーを完了またはスキップ後、再起動しても再表示されない
9. ウォークスルー Step1 の `位置情報を有効化` で位置許可ダイアログが起動する
10. ウォークスルー Step3 の `目的地を開く` で目的地シートが開く
11. Home 上部に `今日の1本` 提案カードが表示され、`このルートで開始` でプレビューへ遷移する
12. `今日の1本` は `10.0〜50.0km` の候補のみ表示され、該当候補がない場合はカード自体が非表示になる
13. Home に `週次ミッション` カードが表示され、距離進捗が更新される
14. 当日の走行距離が増えると Home に `休憩スポット提案` カードが表示され、`休憩ルートを準備` / `このスポットへ案内` でプレビューに遷移する
15. ナビ終了時に `ライドハイライト` シート（距離/平均速度/バッジ）が表示される
16. Home の `フリーライドを開始` から、プレビューなしで記録画面に遷移する
17. フリーライド中は `フリーライド中` と表示され、`目的地再設定` や到着演出が出ない
18. フリーライド終了後、ログ一覧と詳細に `フリーライド` ラベルが表示される
19. iOS設定の `RideLane > 言語` に言語選択が表示され、日本語/英語を選択できる
20. RideLane の言語を日本語に変更後、アプリ再起動でナビ中のルート案内文が日本語寄りに変わる（地図データ依存で英語地名が残る場合あり）
21. 目的地設定を開き、query未入力時に 10〜100km 圏内のおすすめ候補が表示される
22. 目的地設定でテキスト検索を入力すると 200km 圏内で候補が表示される
23. 目的地設定画面でルートモード（`平坦優先` / `ヒルクライム`）を切替できる
24. 目的地候補行の `☆` でお気に入り保存/解除ができる
25. 目的地設定画面の先頭に `お気に入り` セクションが表示される
26. Home の `お気に入りから出発` カードに最大5件表示され、1タップでプレビューへ遷移する
27. 候補選択後にプレビュー画面が表示される
28. スタートで Driving へ遷移し、案内カードが下部、ズームUIが右上、`終了`/`メニュー` が左上縦アイコンで表示される
29. ナビ中に一定時間放置しても画面が自動スリープ/自動ロックしない
30. 案内カード右上に「速度」「全体道のり」が2段表示され、次操作地点までの距離が右上に表示されない
31. 移動中に全体道のりがリアルタイムで減少し、停止時は速度が `0.0 km/h` に収束する
32. ルート上を進むと通過済み区間のルート線が後方から消える
33. 意図的にルートを外れると自動再ルートされ、短時間で連続再ルートしない
34. 位置情報 OFF 時に追跡状態が「位置情報なし」になる
35. 天気カードに天候・風速（m/s）・風向・路面リスクが表示される
36. 天気カード下部に ` Weather` と `https://weatherkit.apple.com/legal-attribution.html` のリンク表示があり、タップで開ける
37. 降雨が予測される場合「xx分後に雨が降る見込み」と表示される
38. ナビ終了後、Health 許可ダイアログを許可すると Apple Health にワークアウト（自転車）が追加される
39. ナビ開始前のプレビューまたはナビ中で、30〜60分先降雨がある場合に `雨回避アラート` バナーが表示される
40. `回避ルート提案` を押すと平坦優先で再ルートされ、ルートが更新される
41. Home 右上の設定アイコンをタップすると設定シートが開く
42. 設定シートの `利用規約` を開くとアプリ内シートで `https://lp.xerographix.co.jp/ridelane/terms.html` が表示される
43. 設定シートの `プライバシーポリシー` を開くとアプリ内シートで `https://lp.xerographix.co.jp/ridelane/privacy.html` が表示される
44. 法務ページ内で `RideLane トップへ戻る` ボタンが表示されない
45. 一度オンラインで法務ページを開いた後、機内モードで再度開いてもページ内容が表示される（キャッシュ表示）
46. 設定シートの `Health連携について` を開くと、同期データ・利用目的・しないことの説明が表示される
47. `Apple Healthに同期する` をOFFにしてライド終了すると、ログ状態が `Health連携オフ` になる
48. `Apple Healthに同期する` をONにしてライド終了すると、従来どおりHealth同期フロー（許可ダイアログ/同期状態表示）が動作する
49. Home表示時に `home_view` Analyticsイベントが送信される（DebugViewまたはXcodeログで確認）
50. ルートプレビューを開くと `route_preview_open` が送信される（`route_mode`, `distance_km`, `eta_min`）
51. プレビューの「スタート」押下で `nav_start` が送信される（`route_mode`, `distance_km`, `eta_min`）
52. ライド終了後に `ride_complete` が送信される（`distance_km`, `avg_speed_kmh`, `badge_count`, `health_sync_enabled`）
53. 目的地に到着すると、ナビ画面下部に到着メッセージバナーが表示される
54. 到着メッセージは約4秒後に自動で非表示になる
55. ライド詳細の地図で軌跡が始点終点の単純な直線にならず、ルート形状として表示される（短時間ライド含む）
56. 目的地到着時（残距離 `0.0km` 表示付近）に赤い終了ボタンを押さなくても、到着メッセージ表示後に自動でナビ終了し、ライドハイライトが表示される
57. ナビ中に速度4km/h以上で一定時間走行すると、上部に補給リマインド（`xx分ごとに給水 / 約xxxkcal/h`）が表示される
58. 停止中（速度4km/h未満）は補給リマインドが表示されない
59. ナビ中に前方急カーブがあるルートで走行すると、`前方急カーブ注意` バナーが表示される
60. 夜間（19:00〜05:59）に高速走行すると、`夜間注意` バナーが表示される
61. 強風/雨天などの警戒時に速度を上げると、`路面悪化注意` バナーが表示される
62. ライドログ詳細を開くと `ライドストーリー` カードが表示される
63. `共有` ボタンでストーリー要約テキストを共有シートへ渡せる
64. ナビ開始時に音声で開始案内が流れる
65. 次操作地点の手前で `300メートル先` と `まもなく` の音声案内が流れる
66. 危険アラート表示時に `危険です` を含む音声警告が流れる
67. 到着時に到着メッセージが音声でも再生される
68. バックグラウンドへ遷移してもナビ中の位置追跡とログ記録が継続する
69. ルート未変更時に `ルートを更新しました` の音声が連発しない
70. バックグラウンド音楽再生中に音声案内が入った後、音楽音量が元へ戻る

## 5. ログ確認
- ルート失敗時に警告文が表示されること
- 逸脱検知時に再ルート開始ログが出ること（DEBUG）
- 速度補完が使われたときに速度更新ログが出ること（DEBUG）
- API 失敗時に「気象データ更新に失敗」が表示されること
- `GoogleService-Info.plist` 未同梱時は `Firebase Analytics disabled` ログが出ること（DEBUG）

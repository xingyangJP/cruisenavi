# 休憩所レコメンド仕様（お気に入り機能と分離）

## 1. 目的
- `お気に入り（任意保存）` と `休憩所レコメンド（自動提案）` を明確に分離する。
- ナビ体験を阻害しないよう、休憩提案は「必要なタイミングで最小限」に出す。

## 2. 用語定義
- お気に入り:
  - ユーザーが任意で保存する固定リスト。
  - 距離制限は設けない（表示上の注意喚起は別途可）。
- 休憩所レコメンド:
  - 走行コンテキストに応じた自動提案。
  - 目的地（ゴール）設定後のみ有効。

## 3. 表示条件（休憩所レコメンド）
- ゴール設定済みであること。
- ルート総距離が `15km以上`。
- ナビ中、またはナビ開始前プレビュー中であること。

## 4. 提案件数
- 通常表示: `1件`（迷わせない）。
- `もっと見る` 操作時: `最大3件` まで。

## 5. 距離帯ロジック（総距離ベース）
- `15–30km`: 休憩提案 `1回`（中盤）。
- `30–60km`: 休憩提案 `2回`（1/3・2/3）。
- `60km以上`: 休憩提案 `3回`（約20kmごと）。

## 6. 候補選定ルール
- 基本探索範囲: 現在地から `2〜8km`（可変）。
- 優先カテゴリ:
  - コンビニ
  - カフェ
  - 公園/休憩所
  - 自販機
- 除外条件:
  - ルートからの乖離が大きい（例: `300m超`）
  - 進行方向と逆方向
  - 既に提案済み、または通過済みスポット

## 7. 気象補正
- 高温・強風・雨予兆がある場合、休憩提案を前倒しする。
- 目安: 次回休憩までの残距離を `20%短縮`。

## 8. UI仕様
- カード表示:
  - タイトル: `あとXkmで休憩推奨` または `休憩スポット提案`
  - 本文: `候補名 / 距離 / ETA`
  - 主CTA: `この休憩所を経由`
  - 副CTA: `別候補`（最大3件内でローテーション）
- 重要:
  - お気に入りセクションとは別ブロックで表示し、混同を防止する。

## 9. 実装指針（最小）
- ViewModel:
  - `activeDestination != nil` でない場合は休憩提案を `nil`。
  - `routeSummary.totalDistance < 15` なら提案しない。
- View:
  - `お気に入り` と `休憩所レコメンド` を別セクション化。
  - 提案カードは1件表示を標準とし、詳細展開時のみ3件表示。

## 10. テスト観点
- ゴール未設定時に休憩提案が表示されない。
- `15km未満` で表示されない。
- `15km以上` で表示される。
- 休憩候補が極端な遠距離（例: 100km超）にならない。
- お気に入り表示と休憩提案表示が混ざらない。

# RideLane ウォークスルー施策（DAU向上）

## 目的
- DL直後ユーザーを「初回ナビ完了」まで最短で到達させる
- 24時間以内の再訪理由を作り、D1 / D7 継続率を上げる
- DAUを増やすため、日次で開く価値（天気・ミッション・進捗）を明確化する

## 北極星指標（North Star）
- `初回起動から24時間以内にナビ開始した率`

## 主要KPI
- Activation: 初回起動 → 目的地設定完了率
- Activation: 初回起動 → ナビ開始率
- Retention: D1 / D7 継続率
- Engagement: 1日あたりのナビ開始回数、天気カード閲覧率、ログ詳細閲覧率

## 対象ユーザーの最短成功体験
1. 現在地が地図に表示される
2. 目的地を1件選ぶ
3. ナビを開始して3分以上走る
4. ライドログを確認する

## ウォークスルー構成（初回起動）
### Step 0: 許諾導線
- 目的: 位置情報許可を取得
- UI: 全画面1枚（スキップ可）
- 文言例: 「現在地から安全な自転車ルートを案内します」
- CTA: `位置情報を有効化`

### Step 1: ホームの価値提示（5秒）
- ハイライト: 地図 / 天気 / ナビ開始ボタン
- 文言例: 「まずは目的地を設定して、最短で1本走ってみましょう」
- CTA: `目的地を設定`

### Step 2: 目的地設定ガイド
- 検索候補の先頭に「はじめてのおすすめ（10〜30km）」を表示
- 1件選択で即プレビューへ遷移
- 離脱対策: 検索0件時に「人気スポットを表示」CTA

### Step 3: ルートモード選択（軽量）
- 2択だけ表示: `平坦優先` / `ヒルクライム`
- 初回デフォルト: `平坦優先`
- 文言例: 「最初は平坦優先がおすすめ」

### Step 4: ナビ開始前の一言
- チェック項目: 距離 / ETA / 雨予測
- CTA: `このルートで開始`

### Step 5: ナビ中の最小チュートリアル（非モーダル）
- 3つの吹き出しだけ表示（各1回のみ）
- 内容:
  - 右上: `速度 / 残り距離`
  - 左上: `終了 / メニュー`
  - 下部: `案内カード（次の操作）`

### Step 6: 完了リワード（終了時）
- 表示: 走行距離、時間、平均速度
- CTA:
  - `今日のログを見る`
  - `明日のおすすめルートを見る`

## 初回以降（DAU化）の日次導線
### 毎日開く理由を固定
- ホーム上部に `今日の提案` を1件表示
  - 例: 「今日は風が弱いので平坦15kmがおすすめ」
- 週次ミッションカード
  - 例: 「今週あと12kmで達成」
- 雨回避バナー（30〜60分先）
  - 雨予測がある日だけ表示

### リマインド（Push）方針
- 初回ナビ未完了ユーザー:
  - DL+24h: 「3分だけ試走してログを作成」
- 初回ナビ完了ユーザー:
  - 翌日朝: 「昨日の続き、あと1本で週目標」
- 7日休眠:
  - 「あなた向けの短距離ルートを作成しました」

## 実装優先順位（小さく始める）
1. 初回ウォークスルー（Step 0〜4）
2. ナビ中チップ（Step 5）
3. 完了リワード（Step 6）
4. 日次提案カード
5. 週次ミッションカード

## A/B テスト案
- Test A: ウォークスルーあり vs なし
  - 主要評価: 初回ナビ開始率
- Test B: 目的地候補の並び
  - `近距離優先` vs `人気優先`
- Test C: 完了画面CTA
  - `ログを見る` vs `次のルート提案`

## イベント計測（最低限）
- `onboarding_started`
- `location_permission_granted`
- `destination_selected`
- `route_mode_selected`
- `navigation_started`
- `navigation_completed`
- `ride_log_opened`
- `daily_suggestion_tapped`

## 受け入れ条件
- 新規ユーザーの80%以上がウォークスルー Step 4 まで到達
- 新規ユーザーの初回ナビ開始率が現状比 +20% 以上
- D1 継続率が現状比 +10% 以上

# Free Cycling Mode Plan

## 1. 目的
- 目的地を設定しないままライドを開始できるモードを追加する
- 通勤、朝練、散歩ライドのような「行き先未定」の走行でも記録価値を作る
- 既存ナビ導線とは別に、記録主体の継続利用導線を用意する

## 2. 想定ユースケース
- 今日は気分で走るので目的地を決めたくない
- 固定ルート周回のトレーニングをしたい
- 通勤/買い物ライドを自動でログ化したい
- 1周ごとのタイムを見たい

## 3. MVP案
- Home に `フリーライド開始` ボタンを追加
- 目的地なしで記録を開始し、位置情報・距離・時間・平均速度を計測
- 終了時に通常のライドログとして保存
- `ラップ` ボタンを追加し、手動ラップを記録
- ラップごとに以下を保存
  - ラップ時刻
  - ラップ時間
  - ラップ距離
  - ラップ平均速度

## 4. UI案
- Home
  - `目的地を設定` の近くに `フリーライド開始`
- 走行中
  - 下部カードを `フリーライド中` 表示へ切替
  - `ラップ` / `終了` の2ボタンを表示
- ライド完了
  - 合計距離、時間、平均速度、ラップ一覧を表示

## 5. データモデル案
- `VoyageLog` にモード種別を追加
  - `destinationNavigation`
  - `freeRide`
- `LapRecord` を追加
  - `timestamp`
  - `elapsedTime`
  - `distanceKm`
  - `averageSpeedKmh`

## 6. 実装方針
- 既存のライドログ保存基盤を流用する
- 目的地未設定時は `activeDestination` / `routeSummary` なしで記録だけ開始する
- Health 同期は通常ライドと同じタイミングで実行可能
- ナビ用の再ルート/到着判定/分岐音声は無効化し、必要なら一定時間ごとの統計読み上げを後続検討にする

## 7. 将来拡張
- 自動ラップ
  - 1kmごと
  - 5分ごと
  - 周回コースの通過点判定
- セグメントベスト
- 心拍/ケイデンス連携
- `今日は何周したか` のストーリー表示

## 8. リスク
- 目的地なしモードでは到着条件がないため、終了忘れが起きやすい
- ラップ追加により走行中UIが複雑化しやすい
- 通常ナビとの状態分岐が増えるので、モード管理を明示的にする必要がある

## 9. 推奨ロードマップ
1. MVP: フリーライド開始/終了、ログ保存
2. ラップ機能: 手動ラップ
3. 詳細強化: ラップ一覧、ベスト表示
4. 自動ラップ/トレーニング分析

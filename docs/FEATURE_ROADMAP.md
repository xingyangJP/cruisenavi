# RideLane 機能企画ロードマップ
更新日: 2026-02-22

## 1. 目的
サイクリングユーザーの「便利さ」と「継続して使いたくなる楽しさ」を両立するため、追加候補機能を優先度付きで整理する。

## 2. 候補機能一覧
| 機能 | 内容 | ユーザー価値 |
|---|---|---|
| 安全ルートスコア | 交通量/坂/夜間/路面状態を元に「安心度」を表示 | 最短だけでなく、走りやすいルートを選べる |
| ヒルクライム/平坦モード | 「獲得標高を抑える」「トレーニングで登る」を切替 | 初心者〜上級者まで目的に合わせて使える |
| 休憩スポット提案 | 走行距離と気温で「あと○kmで休憩推奨」、近くのカフェ/コンビニ提案 | 実走で便利、継続利用につながる |
| 雨回避アラート（30〜60分先） | ルート上の降雨予測を通知し、回避ルートを提案 | 実用性が高く、悪天候時の不安を減らせる |
| セグメントチャレンジ | 区間タイムを自分ベストと比較（他人ランキングなしでも可） | 楽しさとリピート性が上がる |
| 週次ミッション | 「今週40km」「標高500m」などを自動生成 | ゲーム性により継続率を上げる |
| ライドストーリー自動生成 | 走行後に「距離・天気・写真・ハイライト」をカード化 | SNS共有しやすく、達成感を可視化できる |
| グループライドの集合ナビ | 集合地点・遅刻予測・到着通知 | 週末グループライドで使いやすい |
| 補給管理 | 水分/カロリー摂取リマインド | 長距離ライドの安全性・快適性が上がる |
| 危険地点フィードバック | 段差/砂利/狭路などをユーザー投稿 | コミュニティで地図品質を改善できる |

## 3. 優先実装候補（第1フェーズ）
### 3.1 休憩スポット提案
- 理由: 現在の地図・スポット検索機能を再利用しやすく、体験改善効果が大きい。
- MVP案:
  - 走行距離・気温しきい値で休憩推奨を表示
  - 現在地周辺のカフェ/コンビニを3〜5件提示
  - 1タップで目的地再設定に連携

### 3.2 雨回避アラート（30〜60分先）
- 理由: 既存の天気取得基盤を活かして実装でき、実用価値が高い。
- MVP案:
  - ルート上降雨リスクを「低/中/高」で表示
  - 雨見込み時に通知
  - 回避ルート再探索を1タップで実行

### 3.3 ヒルクライム/平坦モード
- 理由: ナビの差別化機能として分かりやすく、ユーザー層を広げられる。
- MVP案:
  - ルート探索時にモード選択を追加
  - 平坦優先: 獲得標高を抑えるルート重み
  - ヒルクライム: 獲得標高を増やすルート重み

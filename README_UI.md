# RideLane UI設計
バージョン: 1.0.58  
更新日: 2026-02-22

## 1. 画面構成
- Home: 現在地地図、天気カード、ログダッシュボード、ナビ開始ボタン
- Destination: おすすめ（10〜100km）/テキスト検索（200km圏内）の目的地検索、ルートモード切替（平坦優先/ヒルクライム）
- Route Preview: ルート全体確認 + スタート/キャンセル
- Driving: 進行方向追従、案内カード（右上: 速度/全体道のり）、ズーム操作、終了/再設定

## 2. デザイン方針
- ライトテーマ（Citrus Road）
- 高コントラストな文字色
- 天気カードは条件連動グラデーションのリッチ表示（ウィジェット風）

## 3. 表示ルール
- 単位は `km` / `km/h`
- 天気の風速単位は `m/s`
- Home 下部に `verX.Y.Z` を中央表示
- 目的地候補は検索モードで範囲を切替（おすすめ: 10〜100km、テキスト検索: 200km圏内）
- Home の現在地地図はカード背景なし・横幅いっぱい（フルブリード）で表示
- Home 地図上部中央に `splash` ロゴを表示
- Home の地図下ステータス（`実GPS` / `GPSアクティブ` など）は表示しない
- iOS設定の `RideLane > 言語` からアプリ表示言語（日本語/英語）を選択できる
- 天気カードは降雨予測が30〜60分先の場合に `雨回避アラート` 文言を表示

## 4. 操作ルール
- ルート開始前に必ずプレビューを表示
- 目的地設定時にルートモード（平坦優先/ヒルクライム）を選択できる
- ナビ中ズームUIは右上固定
- ナビ中の `終了` / `メニュー` は左上にアイコンのみで縦配置
- ナビ案内カードは下部固定表示
- ナビ中は画面の自動スリープを無効化する
- ルート取得失敗時は警告メッセージを表示
- ルート逸脱時は自動再ルートを実行（クールダウンあり）
- プレビュー/ナビ中に30〜60分先の降雨が予測された場合、`雨回避アラート` バナーと `回避ルート提案` ボタンを表示
